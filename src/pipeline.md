# Workflow Pipeline and Visual Analytics for PCA and MDS

[cite_start]This dashboard is designed to evaluate dimensionality reduction techniques by comparing the "relationship phase" (original data) with the "mapping phase" (2D projections)[cite: 11, 41, 70, 71, 72].

## 1. Workflow Pipeline (Data & Logic)

* **Step 1: Calculation of Original Relationships (Backend)**
  * Load the high-dimensional dataset.
  * Calculate the Euclidean distance matrix between all points.
  * Convert distances into attractive weights using a Gaussian Kernel or by selecting the k-nearest neighbors.
  * [cite_start]Calculate Precision (False Positives), Recall (False Negatives), and F-score for each single point based on the original labels[cite: 122, 123, 124, 130, 150, 255, 256].
  * [cite_start]Calculate the Global F-score for the Initial Assessment[cite: 262, 263].

* **Step 2: Projection (Backend)**
  * Execute PCA and store the 2D coordinates.
  * Execute Euclidean MDS and store the 2D coordinates.

* **Step 3: Data Injection (Frontend)**
  * Pass a structured JSON file to the frontend where each point contains: ID, PCA coordinates, MDS coordinates, original label, Precision, Recall, and F-score.

## 2. Web Layout Structure (Single Screen)

To keep everything visible on a single screen (following Visual Analytics principles), a CSS Grid-based layout divided into 4 functional blocks is recommended:

* **Header (Top - 10% Height):**
  * Project title and general controls (e.g., dataset selector).
  * **Initial Assessment:** A visual indicator showing the Global F-score of the original dataset.

* **Control Panel (Left Sidebar - 20% Width):**
  * **Color Scale Selector:** Radio buttons to decide what to color: "Original Labels", "False Positives (Precision)", "False Negatives (Recall)", or "Combined Error (F-score)".
  * **Legend:** The color gradient bar to interpret the values (e.g., Blue/Red for the falsity index).

* **Main Stage (Center - 60% Width):**
  * Two large side-by-side squares (50/50).
  * Left: PCA Scatterplot.
  * Right: Euclidean MDS Scatterplot.

* **Live Analytics (Right Sidebar - 20% Width):**
  * A dynamic panel, empty by default, that populates with bar charts or textual statistics when the user interacts with the points.

## 3. Interactions and Live Analytics (UI/UX)

[cite_start]Coordinated interactions allow users to visually reveal so-called "projection artifacts"[cite: 445, 595].

* **Synchronized Hover:**
  * *Action:* The user hovers the mouse over a point in the PCA scatterplot.
  * *Effect:* The corresponding point in the MDS scatterplot simultaneously enlarges or highlights.

* **Analytical Tooltip (on Hover):**
  * *Action:* Hover over a point.
  * *Live Analytics:* A small box appears showing the actual label of the point, its exact Precision and Recall values, and a brief diagnosis.

* **Brushing and Linking (Area Selection):**
  * *Action:* The user clicks and drags to draw a rectangle (brush) around a visual cluster in the PCA.
  * *Effect:* The selected points are also highlighted in the MDS plot. All other points fade out.
  * *Live Analytics:* The panel instantly calculates the average F-score *only* of the selected points to understand if the visual cluster is reliable or a mapping artifact.

* **Click to Inspect Neighborhood:**
  * *Action:* Single click on a point.
  * [cite_start]*Effect:* The true high-dimensional neighbors of that point (its "True Positives") are highlighted[cite: 124, 128].
  * *Live Analytics:* Visually shows how many of these true neighbors ended up far away in the map, revealing in real-time the False Negatives generated by the projection.
